To prevent test data leakage, you need to submit your model or metric through google forms, and we will run the test results for you. Model Submission: Google Form Steps ðŸªœ
  1. Get the data for the task (identification or generation) that you want to work on from the home page. For identification, you need to work on both loose and strict identification tasks.
  2. For identification: The test data has two columns named sentence1 and sentence2. Your model should output whether sentence2 is a paraphrase of sentence1 as 1 (paraphrase) or 0 (not). Save your predictions in a csv file named {loose/strict}_predictions.csv with the "sentence1", "sentence2" and "prediction" columns.
    For generation: The test data has one columns named sentence1. Your model should generate a paraphrase for the sentence1. Save your generations in a csv file named generations.csv with the "sentence1" and "generation" columns.
  3. Create a zip of your prediction/generation csv files, and name it as submission.zip.
  4. Upload this zip and fill out some additional information in the Google Form.
Metric Submission: Google Form Steps ðŸªœ
  1. You should write a script named run_metric.py that takes the following arguments data_path and output_path. data_path is a csv file containing three columns named sentence1, references, and generation. references is a list of paraphrases for the sentence1. Your script should calcualte your metric score for each generation and save the scores in a csv file named output_path with sentence1, references, generation and score columns.
  2. Upload your run_metric.py and fill out some additional information in the Google Form.
